# Proposal — Blindfold Chess 盲棋训练平台

> 本文档是产品需求提案，明确产品目标、功能边界和优先级。  
> 所有功能设计须经用户确认后方可实现。

---

## 1. 产品定位

**Blindfold Chess** 是一个面向棋手的**盲棋训练平台**，帮助用户在无法看到棋盘的情况下，通过心算和记忆能力进行对弈和练习。

核心价值：
- 提供从入门到进阶的盲棋训练工具
- 降低盲棋练习的门槛（不需要找人对弈，随时随地练习）
- 通过趣味化的练习模式保持用户动力

---

## 2. 目标用户

**主要用户**：有一定国际象棋基础、希望提升盲棋能力的棋手
- 了解基本规则（棋子走法、记谱法）
- 能在有棋盘的情况下正常对弈
- 想要挑战"不看棋盘下棋"

**次要用户**：完全不了解棋的人
- 提供"入门指南"作为可选路径
- 通过坐标练习等简单模式降低认知门槛
- 随时可以"幽默退出"

---

## 3. 功能清单（按优先级排序）

### P0 — 盲棋对战（vs Stockfish）
- 用户选择难度（Easy / Medium / Hard / Expert）
- 与 Stockfish 引擎进行盲棋对弈
- 用户输入代数记谱法（如 e4, Nf3, O-O）
- 可选显示/隐藏棋盘（用于验证或初学辅助）
- 走法历史记录
- 终局判定（将杀、逼和、和棋）

### P1 — 坐标练习（已实现 ✅）
- **模式A（找格子）**：系统随机喊出坐标，用户在棋盘上点击对应格子
- **模式B（报坐标）**：系统高亮一个格子，用户输入坐标
- **计时挑战**：1 分钟内答对越多越好
- 答错后棋盘震动，必须点击正确格子才能进入下一题
- 答错后正确格子显示**绿色脉冲高亮**，错误点击显示**红色闪烁**
- 点击格子有 **scale(0.88) 缩放反馈**
- 实时得分统计
- 全屏大棋盘（48px 方格），方便点击操作
- 可选择执白视角或执黑视角

### P2 — 棋子记忆练习
- 显示一个局面，走几步棋（带动画），然后遮住棋盘
- 询问某个特定格子当前是什么棋子 / 什么颜色
- 考验用户对走子后局面变化的追踪能力

### P3 — 盲棋复盘
- **方式1**：用户粘贴 PGN 文本，网站只显示走法列表，不显示棋盘，用户在脑中推演
- **方式2**：用户逐条输入走法，网站判断是否正确
- **方式3**：内置经典名局，用户选择后"盲着 replay"

## 4. 页面流程

```
[品牌感欢迎页]
    │
    ├── [盲棋练习] → 难度选择 → 对弈界面
    │
    ├── [坐标练习] → 选择白/黑 → 练习界面（A/B模式可选）
    │
    ├── [记忆练习] → （P2 实现后补充）
    │
    ├── [盲棋复盘] → （P3 实现后补充）
    │
    └── [入门指南] → 基础知识介绍 → 可跳转至练习

全局：右上角设置面板（主题切换、语言切换）
```

## 5. 交互与体验设计

### 5.1 退出机制（幽默感）
- 采用**混合风格**，每次退出随机显示一句不同风格的台词
- 风格包括但不限于：温和礼貌、自嘲搞笑、戏精夸张
- 让用户在放弃时也能会心一笑

### 5.2 反馈设计
- 坐标练习答错：**棋盘 shake 震动动画** + 正确格子**绿色脉冲高亮（correctPulse）**
- 坐标练习错误点击：**格子红色闪烁（wrong-tap）**
- 坐标练习点击：**格子 scale(0.88) 缩放反馈**
- 走法错误输入：**输入框红色闪烁**
- 引擎加载失败：**明确提示 + 不启用输入框，防止永久禁用**

### 5.3 棋盘显示
- 坐标练习：**全屏大棋盘**（48px 方格），方便点击
- 盲棋对战：默认隐藏棋盘，用户可手动切换显示
- 统一使用 Grey 主题（浅灰 #b0b0b0 / 深灰 #808080）
- 固定 MPChess SVG 棋子风格

## 6. 不做的事（范围边界）

以下功能**明确不做**，防止范围蔓延：

| 功能 | 原因 |
|------|------|
| 在线多人对战 | 需要服务器、WebSocket、匹配系统，复杂度远超当前范围 |
| 用户注册 / 登录 / 排行榜 | 需要后端数据库和用户系统，当前为纯前端项目 |
| AI 教练分析走法好坏 | 需要复杂的引擎评估接口和 chess AI 分析能力 |
| 移动端 App | 项目定位是网页应用，不考虑打包为原生 App |
| 付费 / 订阅 / 广告 | 免费工具，无商业化计划 |

## 7. 技术约束

| 约束 | 说明 |
|------|------|
| 前端技术栈 | 纯 HTML + CSS + JS，不引入框架（React/Vue 等） |
| 外部依赖 | chess.js v0.13.4（本地）、Stockfish 18（CDN） |
| 部署 | GitHub Pages |
| 兼容性 | 支持主流现代浏览器，优先保证移动端可用 |
| 语言 | 中英双语，通过 i18n 系统切换 |
