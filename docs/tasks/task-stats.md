# Task: StatsModule（统计模块）

> 模块状态：规划中  
> 目标：记录对局历史、胜率统计、进步曲线、成就徽章，支持数据导出  
> 设计文档：high-Level Design.md §4.7

---

## 子任务清单

### 统计界面基础
- [ ] **ST-01** 创建统计界面 HTML 结构：导航标签 + 内容区
- [ ] **ST-02** 导航标签：对局历史 / 胜率统计 / 进步曲线 / 成就徽章
- [ ] **ST-03** 调用 `StatsModule.init()` 初始化统计界面

### 对局历史
- [ ] **ST-04** 从 `StorageModule` 读取对局记录列表
- [ ] **ST-05** 对局历史列表展示：日期、难度、结果、步数、时长
- [ ] **ST-06** 对局历史排序：按时间倒序（最新在前）
- [ ] **ST-07** 点击单条记录查看详情（PGN、走法列表）
- [ ] **ST-08** 对局历史分页或滚动加载（记录多时不卡顿）
- [ ] **ST-09** 删除单条记录功能

### 胜率统计
- [ ] **ST-10** 计算总胜率（胜 / 负 / 和 / 总场数）
- [ ] **ST-11** 按难度统计胜率（Easy / Medium / Hard / Expert 分别计算）
- [ ] **ST-12** 胜率可视化：饼图或柱状图（纯 CSS/JS 实现，不引入图表库）
- [ ] **ST-13** `StatsModule.getWinRate(difficulty?)` 接口实现

### 进步曲线
- [ ] **ST-14** 指标 1：对局总时长趋势（最近 N 场对局的时长变化）
- [ ] **ST-15** 指标 2：平均生存步数趋势（最近 N 场对局的步数变化）
- [ ] **ST-16** 指标 3：各难度胜率变化（随时间的胜率波动）
- [ ] **ST-17** 曲线图纯 JS/CSS 实现（折线图，支持悬停查看数值）
- [ ] **ST-18** `StatsModule.getProgressData()` 接口实现
- [ ] **ST-19** 时间范围筛选：最近 7 天 / 30 天 / 全部

### 坐标练习成绩
- [ ] **ST-20** 从 StorageModule 读取坐标练习历史记录
- [ ] **ST-21** 展示坐标练习最佳成绩、平均正确率
- [ ] **ST-22** 坐标练习正确率趋势图

### 数据导出
- [ ] **ST-23** `StatsModule.exportData('json')`：导出全部数据为 JSON 文件下载
- [ ] **ST-24** `StatsModule.exportData('pgn')`：导出对局记录为 PGN 文件下载
- [ ] **ST-25** 导出按钮 + 文件名自动生成（含日期）

### 数据清空
- [ ] **ST-26** "清空所有数据"按钮（带二次确认弹窗）
- [ ] **ST-27** `StatsModule.clearData()` 清空本地所有统计数据

### 成就徽章系统
- [ ] **ST-28** 徽章数据结构定义：`{ id, name, description, condition, unlocked, date }`
- [ ] **ST-29** 徽章「第一步」：用户完成任意一步走子即可获得
- [ ] **ST-30** 徽章「初尝败绩」：用户输掉第一局
- [ ] **ST-31** 徽章「十步之遥」：单局存活超过 10 步
- [ ] **ST-32** 徽章「首胜」：用户赢得第一局
- [ ] **ST-33** 徽章「连败不屈」：连续输掉 5 局后仍继续
- [ ] **ST-34** 徽章展示界面：已解锁高亮，未解锁灰色锁定
- [ ] **ST-35** 获得新徽章时弹出提示动画

### 事件监听
- [ ] **ST-36** 注册 `BlindfoldModule.onGameOver()` 回调，自动记录对局结果
- [ ] **ST-37** 注册坐标练习结束回调，自动记录练习成绩

### 测试与优化
- [ ] **ST-38** 测试对局记录是否正确存储和读取
- [ ] **ST-39** 测试胜率计算是否准确
- [ ] **ST-40** 测试导出文件格式是否正确
