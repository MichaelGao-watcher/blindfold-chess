# Task: ReplayModule（盲棋复盘模块）

> 模块状态：规划中  
> 目标：支持 PGN 复盘、逐条输入验证、内置经典名局盲着推演  
> 设计文档：high-Level Design.md §4.4

---

## 子任务清单

### 复盘界面基础
- [ ] **R-01** 创建复盘界面 HTML 结构：输入区 + 走法列表区 + 棋盘显示区 + 控制栏
- [ ] **R-02** 三种复盘方式入口：粘贴 PGN / 逐条输入 / 内置名局
- [ ] **R-03** 调用 `ReplayModule.init()` 初始化复盘界面

### 方式一：PGN 粘贴解析
- [ ] **R-04** PGN 文本输入框（支持多行粘贴）
- [ ] **R-05** "加载 PGN"按钮：调用 `ReplayModule.loadPgn(pgnText)`
- [ ] **R-06** PGN 解析器：提取标签（Event, White, Black, Date 等）
- [ ] **R-07** PGN 解析器：提取走法序列，转换为内部 move 列表
- [ ] **R-08** 解析错误提示：格式错误时显示友好提示信息
- [ ] **R-09** 走法列表展示：带序号、可点击跳转

### 方式二：逐条输入验证
- [ ] **R-10** 单步走法输入框
- [ ] **R-11** "验证"按钮：调用 `ReplayModule.verifyMove(input)`
- [ ] **R-12** 与 chess.js 对比验证输入走法是否符合 PGN 记录
- [ ] **R-13** 正确时：更新局面，自动进入下一步
- [ ] **R-14** 错误时：红色提示，不推进局面，允许重新输入
- [ ] **R-15** 支持键盘快捷键（Enter 提交，← → 导航）

### 方式三：内置经典名局
- [ ] **R-16** 从 Lichess 公开数据库获取 100 个经典对局 PGN
- [ ] **R-17** 名局数据打包为 `data/games.js`（本地静态资源）
- [ ] **R-18** 名局列表界面：显示对局名称、棋手、年份、开局名称
- [ ] **R-19** 名局搜索/筛选功能（按棋手、开局、年份）
- [ ] **R-20** 点击名局加载：调用 `ReplayModule.loadClassicGame(id)`

### 走法导航
- [ ] **R-21** 走法列表点击跳转：调用 `ReplayModule.navigateToMove(index)`
- [ ] **R-22** 键盘导航：← 上一步，→ 下一步
- [ ] **R-23** 控制栏按钮：第一步 / 上一步 / 下一步 / 最后一步
- [ ] **R-24** 当前步高亮显示在走法列表中
- [ ] **R-25** 当前步数 / 总步数显示

### 局面显示
- [ ] **R-26** 可选显示当前步局面（`ReplayModule.toggleBoard()`）
- [ ] **R-27** 盲着推演模式：默认隐藏棋盘，用户可手动显示当前局面
- [ ] **R-28** 局面与走法列表同步更新
- [ ] **R-29** 获取当前步 FEN：`ReplayModule.getCurrentFen()`

### 数据持久化
- [ ] **R-30** 用户导入的 PGN 保存到 StorageModule
- [ ] **R-31** 最近复盘记录列表（方便快速继续）

### 测试与优化
- [ ] **R-32** 测试标准 PGN 格式解析是否正确
- [ ] **R-33** 测试逐条输入验证是否严格匹配
- [ ] **R-34** 测试走法导航是否同步更新局面
- [ ] **R-35** 测试 100 个内置名局加载是否正常
