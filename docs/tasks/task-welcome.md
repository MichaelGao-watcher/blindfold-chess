# Task: WelcomeModule（欢迎页模块）

> 模块状态：规划中  
> 目标：打造品牌欢迎页，提供四个模式入口  
> 设计文档：high-Level Design.md §4.1

---

## 子任务清单

### 基础结构
- [ ] **W-01** 创建 `index.html` 中欢迎页容器（`<section id="welcome-screen">`）
- [ ] **W-02** 编写欢迎页基础 HTML 结构：Logo 区 + 标语 + 四个入口按钮
- [ ] **W-03** 实现屏幕切换机制（欢迎页 ↔ 功能页），支持 `WelcomeModule.navigateTo(mode)`

### 视觉样式（苹果风格）
- [ ] **W-04** 定义全局 CSS 变量（深色/浅色主题色、毛玻璃背景、圆角半径、SF 字体栈）
- [ ] **W-05** 欢迎页布局：垂直居中、按钮网格（2×2 或 1×4 响应式）
- [ ] **W-06** 按钮样式：毛玻璃效果（`backdrop-filter: blur`）、圆角、悬停态
- [ ] **W-07** Logo 与标语样式：大字体、渐变或纯白色

### 动态效果
- [ ] **W-08** 动态背景效果（如渐变流动、粒子、棋盘淡影），确保 60fps
- [ ] **W-09** 按钮入场动画（淡入 + 上移， stagger 延迟）
- [ ] **W-10** 低端设备性能回退（减少/关闭动态背景）

### 交互与导航
- [ ] **W-11** 绑定四个入口按钮点击事件 → 调用 `navigateTo('blindfold'|'coordinate'|'replay'|'guide')`
- [ ] **W-12** 导航时保存当前选择到内存（供后续模块初始化使用）
- [ ] **W-13** 从功能页返回欢迎页的入口（全局返回按钮/导航栏）

### 响应式适配
- [ ] **W-14** 移动端适配：按钮堆叠、字体缩小、触控区域 ≥ 44px
- [ ] **W-15** 平板/桌面端适配：更宽的布局、更大的视觉元素

### 测试与优化
- [ ] **W-16** 手动测试：四个入口是否正确跳转到对应模块
- [ ] **W-17** 性能测试：动态背景在低端设备是否流畅
- [ ] **W-18** 代码清理：移除硬编码样式，统一使用 CSS 变量
