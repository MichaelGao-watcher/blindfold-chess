## 二、输入（待讨论）

"输入"指的是：用户能向系统提供什么信息？系统能接收什么数据？

**目前已知的可能性**：
1. 用户点击按钮选择功能模式（盲棋对战 / 坐标练习 / 入门指南）
2. 用户选择难度（Easy / Medium / Hard / Expert）
3. 用户输入代数记谱法走法（如 e4, Nf3, O-O）
4. 用户在坐标练习中点击棋盘格子
5. 用户在坐标练习中输入坐标（如 e4）
6. 用户粘贴 PGN 文本（盲棋复盘，P3 规划中）
7. 用户切换主题（深色/浅色）和语言（中/英）

**不确定的地方**：
- [ ] 用户是否需要注册/登录？（proposal.md 中已明确"不做"，但如果你改变想法，请告诉我）
- [ ] 用户能否自定义引擎难度参数（比如自己填 Elo 分数，而不只是选四个预设）？
- [ ] 用户能否导入/导出自己的对局记录？
- [ ] 用户能否上传自己的 PGN 文件，还是只能粘贴文本？

## 三、输出（待讨论）

"输出"指的是：系统向用户展示什么结果？提供什么反馈？

**目前已知的功能**：
1. 显示/隐藏棋盘
2. 引擎回应走法
3. 走法历史记录
4. 终局判定结果（将杀/逼和/和棋）
5. 坐标练习的得分统计
6. 答错后的震动反馈和高亮提示
7. 主题和语言切换后的界面更新

**不确定的地方**：
- [ ] 对局结束后，是否保存对局记录供用户回顾？
- [ ] 是否显示用户的胜率统计或进步曲线？
- [ ] 坐标练习结束后，是否显示详细报告（比如哪些坐标最容易错）？
- [ ] 引擎思考时，是否显示它在"想哪几步"（比如显示候选走法）？

## 四、步骤 / 流程（待讨论）

"步骤"指的是：用户从打开网页到完成一次训练的完整流程是什么？

**目前已有的流程**（见 proposal.md）：
```
欢迎页 → 选择模式 → 功能界面 → 训练/对弈 → 结束
```

**不确定的地方**：
- [ ] 欢迎页之后，用户是否必须先登录/注册才能使用？（已规划不做，但需确认）
- [ ] 用户中途退出对局，再次打开时是否恢复上次的对局？
- [x] 对局结束后，默认行为是什么？
  - **决策**：弹出结果卡片 → 卡片上显示完整走法列表（PGN 格式），用户可复制 → 提供"再来一局"按钮
- [ ] 坐标练习支持"计时挑战"（1 分钟），这个时长是否可调？
