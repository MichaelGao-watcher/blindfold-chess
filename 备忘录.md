## 二、技术概念解释（盖房子比喻）

想象你要盖一栋房子：

### HTML（HyperText Markup Language）
** HTML = 房子的骨架和结构**

HTML 决定页面上有什么元素：标题、段落、图片、按钮、输入框等。它只负责"有什么"，不管长什么样。

例子：
```html
<h1>这是标题</h1>
<p>这是一段文字</p>
<button>这是一个按钮</button>
```

### CSS（Cascading Style Sheets）
**CSS = 房子的装修、颜色、布局**

CSS 决定这些元素长什么样：颜色、大小、位置、字体、边距、动画效果等。

例子：
```css
h1 {
    color: red;
    font-size: 24px;
}
button {
    background: blue;
    border-radius: 8px;
}
```

### JS / JavaScript
**JS = 房子的电路和智能系统**

JS 让页面能动起来：点击按钮后发生什么、输入内容后的验证、动画控制、数据计算等。

例子：
```javascript
button.addEventListener('click', () => {
    alert('你点击了按钮！');
});
```

### React
**React = 宜家（IKEA）的模块化家具系统**

React 是 Facebook 开发的 JS 库。它的核心思想是"组件化"——把页面拆成一个个独立的模块（组件），像拼乐高一样组合起来。每个组件有自己的状态和逻辑。

- 优点：代码复用性强，大型项目好维护
- 缺点：需要学习成本，项目配置稍复杂
- 适合：中大型项目、团队协作

### Vue
**Vue = 小米的智能家居套装**

Vue 是国人开发的 JS 框架，和 React 类似也是组件化。但它的设计理念更"渐进式"——你可以只用一小部分功能，也可以完整使用全套工具。语法更贴近传统 HTML，上手相对容易。

- 优点：学习曲线平缓，文档对中文用户友好
- 缺点：生态比 React 略小
- 适合：中小型项目、个人开发者

### 一句话总结

| 技术 | 比喻 | 核心作用 |
|------|------|---------|
| HTML | 骨架 | 定义页面上有什么 |
| CSS | 装修 | 定义长什么样 |
| JS | 电路 | 定义能做什么、怎么动 |
| React | 宜家模块 | 用组件化的方式高效盖大房子 |
| Vue | 小米套装 | 用更简洁的方式盖房子，易上手 |

## 五、怎么确认自己在编辑 index.html？

文件夹里有好几个文件（`.gitignore`、`README.md`、`index.html`、`备忘录.md`），新手容易搞混。

**三种确认方式：**

1. **看左侧文件树**：点击 `index.html` 后，文件名会**高亮显示**
2. **看顶部标签页**：编辑器上方会显示 `index.html` 的标签
3. **看窗口标题栏**：显示 `index.html - blindfold-chess - Visual Studio Code`

**最保险的确认方式**：
- 在文件里改一行字（比如把 `盲棋项目开发中...` 改成 `测试123`）
- 按 `Ctrl+S` 保存
- 双击 `index.html` 用浏览器打开
- 如果浏览器里显示 `测试123`，说明你改对了
